# Консолидация сделок: перенос функций из клиентов в /deals

Дата: 2026-06-18
Статус: одобрено к реализации
Ветка: `feature/deals-kanban`

## Контекст и цель

Вкладка «Сделки» в `/clients` и новый раздел `/deals` (канбан) работают с одним
бэкендом (`/api/deals`) — это дублирующие представления одной сущности. Цель:
довести `/deals` до паритета по функциям и **удалить вкладку «Сделки» из клиентов**.

## Что переносим в `/deals`

1. **«Новая сделка»** — `DealEditDialog` (default export, props `{mode:"create", trigger, onSuccess}`; опции грузит сам).
2. **«Найти в источниках»** — `DiscoverDealsDialog` (named export, props `{trigger, onDealsGenerated}`).
3. **Фильтры**: поиск по названию/описанию, селект клиента, тогглы «отменённые» / «удалённые».

Уже есть в `/deals`: взвешенный прогноз, показ по стадиям (колонки), provenance,
drag&drop-перевод, фильтр «Все/Мои».

## Решения (с заказчиком)

- **Тогглы «отменённые/удалённые» нужны.** При включении соответствующие сделки
  показываются **в колонке своей стадии** (`funnelStageId`), приглушёнными
  (`opacity-60`) с бейджем «Отменена»/«Удалена», и **не перетаскиваются**.
- **Удаление вкладки из клиентов — в этой же итерации.**

## Поведение

### Загрузка данных (`/deals/page.tsx`, RSC)
- `listDeals({ includeCancelled: true, includeDeleted: true })` — грузим все.
- `listDealClientOptions()` — для фильтра по клиенту.
- Передаём в `DealsBoard`: `deals`, `stages`, `currentUserId`, `clientOptions`.

### Доска (`deals-board.tsx`)
- Шапка, строка 1: заголовок + прогноз/счётчик слева; справа — кнопки
  «Найти в источниках» и «Новая сделка» (`onSuccess`/`onDealsGenerated` → `router.refresh()`).
- Шапка, строка 2: сегмент «Все/Мои» + поиск (название/описание) + селект клиента
  + чекбоксы «отменённые» / «удалённые».
- Фильтрация (клиентская, композиция): владелец (`filterByOwner`) → поиск → клиент →
  статус-тогглы.
- В колонку стадии попадают: активные сделки (всегда) + отменённые/удалённые
  (если включён соответствующий тоггл) — по их `funnelStageId`.
- Прогноз и счётчик «открытых» считают **только активные** (как сейчас).
- Терминальная полка «Итоги» — без изменений (считает активные на терминальных стадиях).

### Карточка (`deal-kanban-card.tsx`)
- `useDraggable({ id: deal.id, disabled: deal.status !== "active" })`.
- При `status !== "active"`: `opacity-60`, бейдж «Отменена» (cancelled) / «Удалена»
  (deleted), курсор не grab.

### Удаление из клиентов (`clients/page.tsx`)
- Удалить вкладку «Сделки» (`TabsContent value="deals"`) и второй `TabsTrigger`;
  `Tabs` с единственной вкладкой свернуть до прямого рендера секций «Клиенты»/«Контакты».
- Удалить связанный код: стейт `deals/dealStages/dealClientOptions` и их фильтры,
  `loadDeals`, deal-часть `refreshAll`, `DealStageBucket`, `salesFunnelValue`,
  `dealsByStage`, импорты `DealEditDialog`/`DiscoverDealsDialog`/`DealCard`/
  `dealStageLabel`/`Checkbox` (если больше не нужны).
- Удалить файл `src/components/blocks/deal-card.tsx` (используется только здесь).

## Не входит
- Изменение бэкенда — не требуется (всё на существующих API/функциях).
- Перенос фильтра «Все/Мои» в клиентов — там его и не было.

сценарии (создание, AI-поиск, фильтры, тогглы, отсутствие вкладки в клиентах).
